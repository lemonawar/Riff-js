each()

each( function )

Argument

function
The function to be executed while iterating through the selected DOM Elements. The index of the DOM Element will be passed on as the first parameter of this function.

Return

Riff Object

Description

Iterates through the selected DOM Elements and executes a given function. The index of the current DOM Element is passed on as the first parameter of the given function. They keyword 'this' refers to the DOM Element.

Example

The following is a HTML markup of a unordered list.

<ul>
	<li>foo</li>
	<li>bar</li>
</ul>

The following code iterates through each list item and prints the index number and text through a alert window.

$('li').each(function(index) {
    alert(index + ': ' + $(this).text());
  });

filter()

filter( selector )

Argument

selector
A string to select certain DOM Elements within a set of selected DOM Elements. Selector may be a tag name, class name, or ID. (optional)

Return

Riff Object

Description

Selects DOM Elements from the set of selected DOM Elements that match the selector string. If the parameter is empty or not a string, the previosly selected Riff Object is returned.

Example

$('li').filter(".foo").hide();

Selects the "li" tag Element and executes hide() on a DOM Element which has the class name "foo".

Issue

Currently only id, class name, tag names are valid selctors. CSS selector syntax will be applied.

not()

not( index )

Argument

index
Integer. The index of the DOM Element to exclude from the set of selected DOM Elements.

Return

Riff Object

Description

Excludes the DOM Elements that correspond to the given parameter from the set of selected DOM Elements. If the parameter is an index, the DOM Element of the corresponding index number is excluded from the set of selected DOM Elements, and the remaining set is returned.

Example

The following is a HTML markup of a unordered list.

<ul>
	<li id="bar1">foo1</li>
	<li id="bar2">foo2</li>
	<li id="bar3">foo3</li>
</ul>

The DOM Elements that correspond to the selector string are excluded, and the remaining DOM Elements are returned.

var t = $("li").not("#bar1").text();
alert("text:" + t );

"foo2foo3" is printed on the alert window.

not( selector )

buffer()

buffer( key, value )

Argument

key
The identifying string for the "value".
value
The value of the data to store in the buffer

Return

Riff Object

Description

Allocates storage space for each DOM Element that buffer() is executed on, and stores "value" on "key".

The "key" and "value" are stored on riffGlobal.buffer (global array). For every DOM Element that buffer is set, a class designated by the framework is added to the DOM Element.
If there are more than one selected DOM Elements, "value" is stored on "key" for each DOM Element.

Example

The following code stores "123" to "key" to a DOM Element where the ID is "index".

$("#index").buffer("key", 123);

buffer( key )

Argument

key
The identifying string for the "value".

Return

Stored value ( numbers, characters, objects, functions etc )

Description

Finds and returns the "value" which corresponds to the "key" from riffGlobal.buffer (global array).
If there are more than one selected DOM Elements, only the "value" of the first selected DOM Element is returned.

Example

The following code returns the value of "key" rom the DOM Element where the ID is "index".

$("#index").buffer("value");

swipe( isFlickEnd , verlocityPercent, fnEventHandle )

Argument

isFlickEnd
A boolean value to check if the swipe object goes over the wrapping box object. The swipe object cannot go over the box object if the value is true.
verlocityPercent
The movement speed of the swipe event in percentage.
fnEventHandle
The event handler function to run when the event occurs.

Return

Riff Object

Description

The swipe event is a combination of the drag, flickRight, flickLeft events. The parameter of the event handler are as following
_fnEventHandler( state, ptx, pty )
state: flag to distinguish the current state of the drag event
"leftMove" : flag to distinguish when the swipe event occurs and starts moving to the left.
"rightMove" : flag to distinguish when the swipe event occurs and starts moving to the right.
"move" : flag to distinguish when the swipe event occurs and moves.
"leftEnd" : flag to distinguish when the swipe event occurs and moves to the left end of the screen.
"rightEnd" : flag to distinguish when the swipe event occurs and moves to the right end of the screen.
"end" : flag to distinguish the end of the swipe. ptx: the left coordinate of the DOM Element on the swipe event.
pty: the top coordinate of the DOM Element on the swipe event.

Therefore, if a swipe event occurs from left to right, the state values change as the following:
"dragMove" -> "dragMove" -> ... "dragMove" -> "rightMove" -> "move" -> "move" ... "move" -> "rightEnd" -> "end"

xml()

xml(url, fnSuccess, options)

Argument

url
URL to recieve XML feed from
fnSuccess
callback function to run when communication has successfully ended.
options
Options that define communication exceptions and methods.

Return

boolean

Description

Attempts data communication to the designated URL.
If successful, the callback function, fnSuccess(), is executed. If not, the callback funtion designated in options is executed.
The data recieved from the communication is passed on to fnSuccess() as an argument.
    The parameters of fnSuccess are as following.
    fnSuccess( _this, _xml , _xmlText )
        _this: The Riff Object on which the callback was executed (this).
        _xml: If data communication is successful, the XML object of the recieved data is passed on as argument.
        _xmlText: If the recieved XML data is in text, it is passed on as _xmlText
Users may manipulate data as they wish by using _this( = Riff ) and the recieved XML data (xml, xmlText).


The options argument is an object used for AJAX communication options (timeout duration, retry numbers etc) and callback funtions in case of failiure.

The syntax for options is { optionName1 : optionValue1, optionName2 : optionValue2, ... }. The following options are available.

options = {
    e: fnError,
    a: fnAbort,
    r: fnRetry,
    t: fnTimeout,
    timeout : 5000,
    retry : 1,
    type: "GET",
    isAsync: true,
    sendData:"userValue=uservalue1"
}

e: The callback funtion to execute in case of error. For example, if { e: fnError }, then fnError is the callback function.
    The parameters of fnError() are as following.
    fnError( _readyState, _status )
        _readyState : The readyState of the xHttpRequest object.
        _status : The communication status of the xHttpRequest object.
    If any of the following critera are met, the communication is considered as failure.
        Communication failed ( readyState != 4 ), and timeout has expired.
        Retry value is not designated, data communication is successful, but communication status is not normal ( status != 0 or 200 )
        Retry value is not designated, data communication is successful, communication status is normal ( status = 0 or 200 ), but responseXml = null and responseText = null.
        Retry value is designated, and the number of retries exceeds the designated retry number.
        Retry and timeout value are designated, but the number of retries due to timeout exceeds the designated retry value.
a: The callback funtion to execute in case of communication abort. For example if { a: fnAbort }, then fnAbort is the callback function.
    The parameters of fnAbort() are as following.
    fnAbort( _this, _retryCount )
        _this : The AJAX object that is communicating.
        _retryCount : The number of retries.
r: The callback funtion to execute when data retransmission occurs. For example if { r: fnRetry }, then fnRetry is the callback function.
    The parameters of fnRetry() are as following.
    fnRetry( _this, _retryCount )
        _this : The AJAX object that is communicating.
        _retryCount : The communication retry count.
t: The callback funtion to execute when data commmunication does not succeed within the designated timeout. For example if { t: fnTimeout }, then fnTimeout is the callback function.
    There are no parameters for fnTimeout.
timeout : The maximum time to wait for data communication in milliseconds. For example, if { timeout : 5000 } then timeout is set to 5 seconds.
retry : The maximum number of retries. For example, if { retry:1 }, only one retry is attempted in case a exception occurs. If communication is successful fnSuccess() is executed, but in case of an failure, excetptions are processed as explained above.
type : Data communication type. A data communication type may be either "GET" or "POST".
isAsync : Data communication method, asynchronous or synchronous. For example, if { isAsync : true } then, data is exchanged asynchronously.
sendData : The data to transmit. For example, if { sendData:"userValue=uservalue1" } then, the data userValue=uservalue1 is transmitted, and the result is recieved.

The default option values are as following.

e: Unless specified by the user, xHttpRequest.readyState, xHttpRequest.status is displayed by the Riff.alert() function.
    They following string shows on the alert window.
    "Error occured: readyState:" + readyState + " status:" + status + " " + error information;
a: Unless specified by the user, does nothing.
r: Unless specified by the user, does nothing.
t: Unless specified by the user, does nothing.
timeout: Unless specified by the user, default timeout value is 5000. If timeout value is 0, then there is no timeout.
retry: Unless specified by the user, the default retry value is 1. If retry value is 0, then there is no retry.
type : Unless specified by the user, the default type value is "GET".
isAsync : Unless specified by the user, the default isAsync value is true (asynchronous).
sendData : Unless specified by the user, no data is transmitted.
